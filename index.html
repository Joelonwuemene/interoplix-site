Case study

Multi-site rollout at scale.

A representative engagement from the team's delivery record. Client named generically for confidentiality.

Global diagnostics manufacturer

Standardising device integration across 30+ facilities in 8 countries.

Situation
A global diagnostics manufacturer needed to integrate diagnostic devices across more than 30 healthcare facilities in 8 countries. Each site ran different EHR systems, IT teams, and regulatory requirements, with no standardised integration process. Average deployment time was 8 weeks per site.
Task
Lead a team of 5 integration specialists to deliver every site on time and on budget, compliant with HIPAA, GDPR, and FDA 21 CFR Part 11 at the same time.
Approach
We built standardised HL7 and FHIR integration frameworks and reusable connectivity templates, so each site deployment became a configuration exercise rather than an engineering project. We architected the Azure integration layer using Logic Apps and API Management, giving secure connectivity between on-premises lab systems and cloud platforms.
Outcome
A repeatable rollout model that held its quality bar across every site.

Architecture

How the pipeline works.

End to end: HL7 v2.x ingestion, AHDS $convert-data to FHIR R4, a validation gate that dead-letters bad data before write, persistence in Azure Health Data Services, and de-identified export for analytics. Validation before write, audit at every boundary, reproducible in Bicep.
